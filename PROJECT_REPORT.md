## 1. 项目概述

### 1.1 项目背景

**暖学帮**是一款面向青少年的AI心理关怀与学业辅助系统，基于大语言模型（LLM）+ 检索增强生成（RAG）+ 心理学专业模块，提供：

- **心理状态检测**：情绪识别、危机干预
- **学业辅助**：知识问答、学习方法指导
- **个性化关怀**：基于用户画像的定制化回复

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| **LLM** | 通义千问 (qwen-max) + MiniMax API |
| **向量数据库** | ChromaDB |
| **Embedding** | LangChain + 千问Embedding |
| **后端框架** | Flask |
| **Python版本** | 3.11 |
| **容器化** | Docker |

### 1.3 核心特性

```
┌─────────────────────────────────────────────────────────┐
│                    核心特性                              │
├─────────────────────────────────────────────────────────┤
│  • 多意图识别 (5种意图类型)                              │
│  • 三层记忆系统 (对话/用户/情感)                        │
│  • 危机干预机制 (4级预警)                              │
│  • 工作流引擎 (6种任务类型)                            │
│  • 多级缓存 (L1/L2/L3)                                 │
│  • 混合检索 (向量+BM25)                                │
│  • 性能监控 (实时指标)                                 │
└─────────────────────────────────────────────────────────┘
```

## 3. 核心模块详解

### 3.1 Orchestrator（主编排器）

**文件**：`agent/core/orchestrator.py`

**职责**：系统的中央调度器，协调所有子模块处理用户请求。

**核心逻辑**：
```
用户消息
    │
    ▼
┌─────────────────────────┐
│  1. 意图路由            │ ← IntentRouter 判断意图类型
│     (Intent Routing)    │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│  2. 并行执行            │
│     • 记忆读取          │
│     • RAG检索           │
│     • 心理学检测         │
│     (Parallel Execute)  │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│  3. LLM生成响应         │
│     (LLM Generation)   │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│  4. 反思机制            │
│     (Reflection)        │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────┐
│  5. 更新记忆            │
│     (Memory Update)     │
└─────────────────────────┘
    │
    ▼
响应
```

**关键配置**：
```python
@dataclass
class OrchestratorConfig:
    name: str = "暖学帮智能助手"

    temperature: float = 0.7
    max_tokens: int = 1500
    enable_rag: bool = True
    enable_tools: bool = True
    enable_psychology: bool = True
    max_workflow_steps: int = 5
```

### 3.2 IntentRouter（意图路由）

**文件**：`agent/core/intent_router.py`

**职责**：判断用户消息的意图类型，决定系统响应策略。

**5种意图类型**：

| 意图类型 | 说明 | 触发关键词 | 系统响应 |
|---------|------|----------|---------|
| **CRISIS** | 危机信号 | 想死、自杀、自残 | 危机干预（最高优先级） |
| **PSYCHOLOGY** | 心理倾诉 | 情绪、压力、焦虑、人际关系 | 心理学模块处理 |
| **EDUCATION** | 学业问题 | 学习、考试、作业、知识点 | RAG检索+知识库 |
| **KNOWLEDGE** | 知识问答 | 是什么、怎么做、为什么 | 通用知识库 |
| **CHAT** | 闲聊 | 其他 | 闲聊模式 |

**路由流程**：
```
用户消息
    │
    ▼
┌─────────────────────────┐
│  危机关键词检测          │
│  (Crisis Keywords)     │
└─────────────────────────┘
    │ 发现危机 → CRISIS
    ▼
┌─────────────────────────┐
│  心理学关键词检测        │
│  (Psychology Keywords)  │
└─────────────────────────┘
    │ 发现心理学 → PSYCHOLOGY
    ▼
┌─────────────────────────┐
│  教育类关键词检测        │
│  (Education Keywords)  │
└─────────────────────────┘
    │ 发现教育 → EDUCATION
    ▼
┌─────────────────────────┐
│  知识问答模式            │
│  (Knowledge Pattern)    │
└─────────────────────────┘
    │ 匹配 → KNOWLEDGE
    ▼
    CHAT (默认)
```

### 3.3 WorkflowEngine（工作流引擎）

**文件**：`agent/core/workflow_engine.py`

**职责**：将复杂请求分解为可执行任务，管理依赖和并行执行。

**6种任务类型**：

| 任务类型 | 优先级 | 说明 | 依赖 |
|---------|-------|------|------|
| `CRISIS_DETECT` | CRITICAL (0) | 危机检测 | 无 |
| `EMOTION_DETECT` | HIGH (1) | 情绪识别 | 无 |
| `RAG_RETRIEVE` | NORMAL (2) | 知识检索 | 无 |
| `LLM_GENERATE` | LOW (3) | LLM生成 | RAG_RETRIEVE |
| `REFLECTION` | LOW (3) | 反思机制 | LLM_GENERATE |
| `MEMORY_UPDATE` | LOW (3) | 记忆更新 | EMOTION_DETECT, LLM_GENERATE |

**执行模式**：
```
任务分解
    │
    ▼
依赖解析 ───────────────────▶ 任务D (依赖A,B)
    │                          │
    ▼                          ▼
就绪任务 ───────────────────▶ 任务A    任务B    任务C (并行执行)
    │                          │         │         │
    ▼                          ▼         ▼         ▼
可执行 ───────────────────▶ [RUNNING] [RUNNING] [RUNNING]
    │                          │         │         │
    │                          └─────────┴─────────┘
    │                                    │
    ▼                                    ▼
后台执行 ──────────────────────────▶ [COMPLETED]
    │
    ▼
结果聚合
```

### 3.4 UnifiedMemoryManager（统一记忆管理）

**文件**：`agent/memory_store/unified_memory.py`

**职责**：管理用户的多层记忆，支持对话历史、用户画像、情感追踪。

**四层记忆架构**：

```
┌─────────────────────────────────────────────────────────┐
│                   统一记忆管理器                         │
│              UnifiedMemoryManager                       │
└─────────────────────────────────────────────────────────┘
          │              │              │              │
          ▼              ▼              ▼              ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  对话记忆    │  │  用户画像    │  │  情感历史    │  │  知识记忆    │
│  Dialogue   │  │  Profile    │  │  Emotion    │  │  Knowledge  │
│  Memory     │  │  Memory     │  │  History    │  │  Memory     │
└─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘
     │              │              │              │
     ▼              ▼              ▼              ▼
  会话级        用户级         用户级         全局级
  (session)    (user_id)     (user_id)     (全局)
     │              │              │              │
     ▼              ▼              ▼              ▼
  短期记忆       长期特征        情绪变化        事实知识
  (可配置TTL)    (持久化)       (趋势分析)      (持久化)
```

**对话记忆**：
```python

# 返回: [{"role": "user", "content": "...", "timestamp": ...}, ...]
```

**用户画像**：
```python
# 创建画像
```

### 3.10 ContextPipeline（上下文流水线）

**文件**：`agent/context/context_pipeline.py`

**职责**：对上下文进行6阶段处理。

**6个处理阶段**：
```
输入 → 提取 → 转换 → 丰富 → 过滤 → 排序 → 聚合 → 输出
```

| 阶段 | 功能 |
|-----|------|
| 提取 (Extraction) | 从原始输入提取关键信息 |
| 转换 (Transformation) | 格式转换、标准化 |
| 丰富 (Enrichment) | 补充元数据、标签 |
| 过滤 (Filtering) | 去除无关、重复信息 |
| 排序 (Ranking) | 按重要性/相关性排序 |
| 聚合 (Aggregation) | 合并相似内容 |

### 3.11 Performance模块（性能监控）

**文件**：`agent/performance/`

**三个子模块**：

| 模块 | 文件 | 功能 |
|-----|------|------|
| `BenchmarkRunner` | `benchmark.py` | 基准测试，测量各模块性能 |
| `Profiler` | `profiler.py` | 函数级追踪，识别瓶颈 |
| `RuntimeMonitor` | `monitor.py` | 实时监控，告警管理 |

**监控指标**：
```
- request_count: 请求计数
- request_latency: 请求延迟
- error_count: 错误计数
- active_sessions: 活跃会话
- cache_hit_rate: 缓存命中率
- rag_retrieval_time: RAG检索耗时
- emotion_detection_time: 情绪检测耗时
- crisis_detection_time: 危机检测耗时
```

**告警规则**：
