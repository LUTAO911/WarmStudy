## 4. 技术实现详情

### 4.1 智能体架构

#### 4.1.1 状态机设计
```python
class AgentState(Enum):
    """智能体状态枚举"""
    IDLE = "idle"                     # 空闲状态
    INTENT_DETECTION = "intent_detection" # 意图检测
    SKILL_ROUTING = "skill_routing"   # 技能路由
    TOOL_EXECUTING = "tool_executing" # 工具执行
    KNOWLEDGE检索 = "knowledge_retrieval" # 知识检索
    RESPONSE_GENERATING = "response_generating" # 响应生成
    CRISIS_HANDLING = "crisis_handling" # 危机处理
    RESPONSE_READY = "response_ready" # 响应就绪
    ERROR = "error"                   # 错误状态
```

#### 4.1.2 工作流程
1. **意图识别**：分析用户输入，识别对话意图
2. **危机检测**：检测是否存在危机内容
3. **知识检索**：从知识库中检索相关信息
4. **工具调用**：执行必要的工具操作
5. **响应生成**：基于大模型生成回复
6. **记忆存储**：保存对话历史到长期记忆

### 4.2 RAG知识库

#### 4.2.1 知识存储
- **向量数据库**：使用ChromaDB存储向量嵌入
- **文件格式**：支持md、txt、pdf、docx、xlsx、csv、json、html
- **文本分块**：500字符/块，50字符重叠
- **向量模型**：使用Qwen Embedding

#### 4.2.2 检索流程
1. **查询向量化**：将用户查询转换为向量
2. **相似性搜索**：在向量数据库中查找相似文档
3. **结果过滤**：过滤低相似度结果（阈值0.3）
4. **结果排序**：按相似度分数排序
5. **知识融合**：将检索到的知识融入对话

### 4.3 大模型集成

#### 4.3.1 模型选择
- **主要模型**：通义千问（Qwen）
- **备用模型**：MiniMax 2.7
- **向量模型**：Qwen Embedding

#### 4.3.2 调用方式
- **API调用**：通过官方API接口调用
- **参数配置**：temperature=0.7，max_tokens=2000
- **错误处理**：实现重试机制和模型切换

### 4.4 前端实现

#### 4.4.1 技术栈
- **开发框架**：微信小程序原生开发
- **语言**：TypeScript + JavaScript
- **UI组件**：原生组件 + 自定义组件
- **网络请求**：wx.request

#### 4.4.2 关键功能
- **实时对话**：WebSocket或轮询方式
- **消息历史**：本地存储聊天记录
- **语音输入**：支持语音转文本
- **情绪识别**：前端情绪展示
