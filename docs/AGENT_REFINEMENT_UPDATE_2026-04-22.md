# 智能体精修更新说明（2026-04-22）

本次更新围绕“学生端心理陪伴智能体 + 家长端支持智能体”做了行为级精修，目标不是改界面风格，而是让智能体真正具备角色区分、年龄适配和实时状态感知能力。

## 本次核心更新

- 新增独立策略引擎：`agent/agent/strategy_engine.py`
- 学生与家长聊天链路分流，不再只共用一套上下文判断
- 学生资料支持同步到后端：姓名、性别、年龄、年级、学段
- 新增学生资料接口：`POST /api/student/profile`
- 学生打卡、测评结果会刷新当前策略状态
- 聊天请求新增 `session_id`、`profile`、`child_id` 等上下文字段
- 前端聊天页和家长聊天页已同步带上这些上下文，但没有改页面视觉风格
- 聊天页与网页后台补充了逐字输出式的回答呈现
- 管理员后台新增模型配置面板，可直接查看和修改聊天模型、RAG 模型、Embedding 模型

## 智能体能力变化

### 1. 角色分流

- 学生端：以情绪承接、陪伴、温和鼓励为主
- 家长端：以解释、观察点、沟通建议、下一步动作建议为主

### 2. 年龄/年级适配

- 根据年龄和年级推断学段：小学、初中、高中、大学/未知
- 不同学段采用不同沟通策略
- 小学：更短句、更具体、更少抽象词
- 初中：更温和、更尊重、自我表达引导更多
- 高中：更成熟、克制、减少说教感

### 3. 实时状态感知

- 学生提交打卡后，后端刷新心理状态快照
- 学生提交测评后，后端刷新最新报告与策略状态
- 家长对话会自动带上当前绑定孩子的状态摘要

## 接口变化

### 新增

- `POST /api/student/profile`

请求体示例：

```json
{
  "user_id": "student_001",
  "name": "小明",
  "gender": "男",
  "age": 13,
  "grade": "初一"
}
```

### 增强

- `POST /api/student/chat`
  - 新增可选字段：`session_id`、`profile`
- `POST /api/parent/chat`
  - 新增可选字段：`session_id`、`child_id`

## 前端同步点

- 学生端资料弹窗提交后，会同步调用 `updateStudentProfile`
- 学生聊天会带当前会话 `sessionId` 与本地学生资料
- 家长聊天会带当前绑定孩子 `childId` 与当前会话 `sessionId`

## 对服务器的影响

如果服务器已经部署完成，这次更新后需要重新构建并重启容器，让新的策略引擎、资料同步接口和聊天上下文字段生效。

建议执行：

```bash
git pull
docker compose up -d --build
```

然后验证：

- `POST /api/student/profile` 返回 200
- 学生聊天返回字段中包含 `strategy`
- 家长聊天返回字段中包含 `strategy`
- 学生提交打卡/测评后，`/api/student/psych/status/{user_id}` 中可看到更新后的 `strategy`
