# WarmStudy Documentation Index

本文档目录用于补足项目当前缺少的交付型说明，重点覆盖部署、模型选型与商业落地。

## 推荐阅读顺序

1. `DEPLOYMENT_RUNBOOK.md`
   适合第一次接手项目的同学，包含当前代码结构、环境变量、启动方式、服务器部署建议与排障要点。
2. `MODEL_SELECTION.md`
   适合需要做模型替换、成本测算、答辩说明的人，包含当前代码支持情况、推荐模型组合与阶段性选型策略。
3. `COMMERCIALIZATION_PLAN.md`
   适合准备比赛答辩、项目路演、校内试点、商业合作时使用，包含目标客户、产品形态、收入模式、实施路径与风险控制。
4. `OPEN_SOURCE_AND_COMPONENTS.md`
   适合比赛提交材料整理，说明开源框架、库、组件、许可证、是否改源码以及是否使用第三方商用模型 API。
5. `MODEL_SOURCE_AND_DEPLOYMENT_NOTE.md`
   适合说明模型来源、API 接入方式、环境变量配置、是否包含本地权重以及现场部署注意事项。
6. `PROJECT_OVERVIEW_FORM.md`
   适合直接整理作品信息概要表。
7. `PPT_AND_DEMO_OUTLINE.md`
   适合准备 3 分钟演示视频与答辩 PPT。
8. `COMPETITION_SUBMISSION_CHECKLIST.md`
   适合赛前逐项核对提交材料是否完整。

## 当前项目说明

- 前端目录：`WarnStudty`
- 后端目录：`agent`
- 当前实际运行方式：
  - `agent/app.py` 提供 RAG/Agent 服务，默认端口 `5177`
  - `agent/api_gateway.py` 提供网关接口，默认端口 `8000`
- 当前代码已支持的模型接入：
  - 阿里云百炼 / 通义千问
  - MiniMax
- 根目录 `docker-compose.yml` 仍带有历史残留配置，部署时优先参考本目录中的运行手册，而不是直接照搬旧文档。

## 文档使用建议

- 如果目标是“先跑通演示”，先看部署手册里的“最小可运行方案”。
- 如果目标是“上线一个校内试点”，同时看部署手册和商业落地方案。
- 如果目标是“控制调用成本”，重点看模型选型文档中的分层路由建议。

## 更新原则

- 涉及价格、限流、模型能力的内容，默认以官方文档为准。
- 涉及合规、采购、医疗/心理服务边界的内容，本文档只提供产品与工程建议，不替代正式法律意见或执业意见。
