# WarmStudy智能心理辅导系统 - 模型权重获取说明

## 1. 模型概述

WarmStudy系统使用了以下AI模型：

| 模型类型 | 模型名称 | 用途 | 获取方式 |
|----------|----------|------|----------|
| 大语言模型 | 通义千问（Qwen） | 对话生成 | API调用 |
| 大语言模型 | MiniMax 2.7 | 对话生成（备用） | API调用 |
| 向量模型 | Qwen Embedding | 文本向量化 | API调用 |

## 2. API密钥获取

### 2.1 通义千问 API 密钥

**获取步骤**：
1. 访问阿里云开发者平台：[https://dashscope.console.aliyun.com/](https://dashscope.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 进入"API密钥管理"页面
4. 点击"创建API密钥"
5. 复制生成的API密钥

**使用方式**：
将API密钥添加到 `.env` 文件中：
```
QWEN_API_KEY=your_qwen_api_key_here
```

### 2.2 MiniMax API 密钥

**获取步骤**：
1. 访问MiniMax官网：[https://www.minimaxi.com/](https://www.minimaxi.com/)
2. 注册/登录账号
3. 进入"开发者中心"
4. 点击"创建API密钥"
5. 复制生成的API密钥

**使用方式**：
将API密钥添加到 `.env` 文件中：
```
MINIMAX_API_KEY=your_minimax_api_key_here
```

## 3. 本地模型部署（可选）

如果需要本地部署模型，可参考以下步骤：

### 3.1 推荐本地模型

| 模型名称 | 下载地址 | 适用场景 |
|----------|----------|----------|
| Qwen2-7B-Instruct | [Hugging Face](https://huggingface.co/Qwen/Qwen2-7B-Instruct) | 对话生成 |
| BAAI/bge-large-zh | [Hugging Face](https://huggingface.co/BAAI/bge-large-zh) | 文本向量化 |
| InternLM2-Chat-7B | [Hugging Face](https://huggingface.co/InternLM/InternLM2-Chat-7B) | 对话生成 |

### 3.2 本地部署步骤

#### 步骤 1：安装依赖
```bash
pip install transformers torch accelerate
```

#### 步骤 2：下载模型
```bash
# 方法1：使用Hugging Face CLI
huggingface-cli download Qwen/Qwen2-7B-Instruct --local-dir ./models/qwen2-7b-instruct

# 方法2：使用transformers自动下载
from transformers import AutoModelForCausalLM, AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2-7B-Instruct")
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2-7B-Instruct",
    device_map="auto",
    torch_dtype="auto"
)
```

#### 步骤 3：配置本地模型

修改 `app/core/llm.py` 文件：
```python
def get_local_chat():
    """获取本地大模型"""
    from transformers import AutoModelForCausalLM, AutoTokenizer
    
    tokenizer = AutoTokenizer.from_pretrained("./models/qwen2-7b-instruct")
    model = AutoModelForCausalLM.from_pretrained(
        "./models/qwen2-7b-instruct",
        device_map="auto",
        torch_dtype="auto"
    )
    
    def generate_response(prompt, max_length=1000):
        inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
        outputs = model.generate(
            **inputs,
            max_length=max_length,
            temperature=0.7,
            top_p=0.9
        )
        return tokenizer.decode(outputs[0], skip_special_tokens=True)
    
    return generate_response
```

## 4. 模型参数配置

### 4.1 大模型参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| temperature | 0.7 | 控制生成文本的随机性 |
| max_tokens | 2000 | 最大生成 tokens 数 |
| top_p | 0.9 | 控制采样范围 |
| frequency_penalty | 0.0 | 控制重复词的惩罚 |
| presence_penalty | 0.0 | 控制新主题的引入 |

### 4.2 向量模型参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| embedding_dim | 1024 | 嵌入维度 |
| batch_size | 32 | 批处理大小 |
| normalize_embeddings | True | 是否归一化嵌入向量 |

## 5. 模型切换

```python
    # 优先使用通义千问
    try:
        return get_qwen_chat()
    except Exception:
        #  fallback 到 MiniMax
        return get_minimax_chat()
```

### 5.2 本地与在线模型切换

修改 `app/core/llm.py` 文件：
```python
def get_chat_model():
    """获取对话模型"""
    # 优先使用本地模型
    try:
        return get_local_chat()
    except Exception:
        # fallback 到在线模型
        try:
            return get_qwen_chat()
        except Exception:
            return get_minimax_chat()
```

## 6. 模型性能评估

### 6.1 评估指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| 响应时间 | 从输入到生成回复的时间 | <2秒 |
| 准确率 | 回复内容的准确性 | >85% |
| 相关性 | 回复与用户问题的相关性 | >90% |
| 安全性 | 避免生成有害内容 | 100% |

### 6.2 评估方法

1. **人工评估**：邀请专业心理咨询师评估回复质量
2. **自动评估**：使用BLEU、ROUGE等指标评估生成质量
3. **用户反馈**：收集用户对回复的满意度评价

## 7. 模型优化

### 7.1 推理优化

- **量化**：使用INT8或INT4量化减少模型大小
- **加速**：使用vLLM或TGI等推理加速框架
- **批处理**：批量处理请求提高吞吐量

### 7.2 微调（可选）

如果需要针对心理健康领域进行模型微调：

1. **准备数据集**：收集心理健康对话数据
2. **微调配置**：
   ```bash
   python -m transformers.TrainingArguments 
       --output_dir ./fine-tuned-model 
       --per_device_train_batch_size 8 
       --learning_rate 2e-5 
       --num_train_epochs 3 
       --warmup_steps 500
   ```
3. **评估微调效果**：使用验证集评估微调后模型的性能

## 8. 注意事项

### 8.1 API使用限制

- **调用频率**：注意各API的调用频率限制
- **token限制**：注意单次请求的token数限制
- **费用**：使用在线API会产生费用，注意监控使用量

### 8.2 模型安全

- **内容过滤**：确保模型不会生成有害内容
- **隐私保护**：处理用户输入时注意隐私保护
- **伦理考量**：确保模型使用符合伦理规范

### 8.3 本地部署要求

- **硬件要求**：7B模型需要至少16GB GPU内存
- **软件要求**：CUDA 11.7+，Python 3.8+
- **存储空间**：7B模型需要约14GB存储空间

## 9. 故障排查

### 9.1 API调用失败

**症状**：模型调用返回错误

**解决方法**：
1. 检查API密钥是否正确
2. 检查网络连接是否正常
3. 检查API调用频率是否超过限制
4. 查看错误信息，根据错误提示进行处理

### 9.2 本地模型加载失败

**症状**：本地模型无法加载

**解决方法**：
1. 检查模型文件是否完整
2. 检查硬件资源是否充足
3. 检查依赖包是否正确安装
4. 查看错误信息，根据错误提示进行处理

### 9.3 生成质量问题

**症状**：模型生成的回复质量不佳

**解决方法**：
1. 调整模型参数（如temperature、top_p）
2. 优化提示词工程
3. 考虑使用更适合的模型
4. 增加领域特定的微调
