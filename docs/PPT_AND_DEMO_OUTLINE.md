# PPT And Demo Outline

This outline is prepared for a 3-minute demo video and defense PPT.

## 1. Recommended PPT Structure

### Page 1. Project Title

- project name
- competition direction
- team name
- one-sentence positioning

Suggested line:

“WarmStudy is an adolescent psychological care agent system for school and family scenarios.”

### Page 2. Pain Point And Scenario

- what problem exists
- why current solutions are insufficient
- why this scenario matters in education

### Page 3. Solution Overview

- student side
- parent side
- school-side extension
- overall value loop

### Page 4. System Architecture

- frontend interaction layer
- API gateway / service layer
- agent layer
- RAG knowledge layer
- external model and tool layer

### Page 5. Core Technologies

- large model
- RAG
- tool calling
- workflow orchestration
- frontend/backend integration

### Page 6. Functional Demo

- student emotional dialogue
- daily check-in
- parent-side support dialogue
- knowledge retrieval
- warning or report support

### Page 7. Innovation Highlights

- education direction and psychological care focus
- low-threshold support path
- student-parent-school loop
- engineering-based prototype

### Page 8. Deployment And Deliverability

- runnable code
- deployment method
- environment configuration
- API model access

### Page 9. Commercial And Social Value

- school trial
- family support
- future SaaS possibility

### Page 10. Summary

- what problem is solved
- what technology is used
- why it is meaningful

## 2. Recommended 3-Minute Demo Flow

### Segment 1. Problem Introduction

Time: 20 to 30 seconds

- introduce adolescent emotional support pain point
- explain why this project is needed

### Segment 2. Student-Side Demo

Time: 60 to 80 seconds

- enter student side
- show dialogue with AI companion
- show check-in or emotional record
- show psychological knowledge entry

### Segment 3. Parent-Side Demo

Time: 40 to 60 seconds

- show parent-side AI consultation
- show trend or report-related information

### Segment 4. Technical Highlight

Time: 30 to 40 seconds

- explain large model + RAG + tool calling
- point to architecture or workflow

### Segment 5. Closing Value

Time: 20 to 30 seconds

- educational value
- social value
- future landing potential

## 3. Defense Preparation Focus

The judges are likely to ask:

1. why this scenario is meaningful
2. where the RAG capability is reflected
3. what tool calling is actually used
4. whether the system can be deployed independently
5. whether the model is open source or API-based
6. what is innovative compared with generic chatbots

## 4. Recommended Short Answer For Tool Calling

Suggested answer:

“Our system is not only a plain dialogue interface. It combines domain knowledge retrieval, state recording, and structured backend interaction through tool-like service calls, so the agent can complete retrieval, record submission, and response generation in one workflow.”

## 5. Recommended Short Answer For Model Access

Suggested answer:

“The current competition version mainly uses API-based large model access for stable demo and lower deployment complexity, while the architecture preserves the possibility of extending to additional providers or local deployment later.”
