# WarmStudy 赛题对齐说明文档

## 1. 项目名称

WarmStudy（暖学帮）：面向青少年心理关怀的教育智能体原型系统

## 术语说明

本项目的前端应用形态建议统一表述为“基于微信小程序技术实现的 App 应用前端”。
项目工程目录 `WarnStudty/` 使用的是微信小程序技术栈，但本项目对外展示和产品定位应理解为 App 应用，而不是普通的小程序演示页面。

## 2. 赛题对应关系

依据 [2026年广东省大学生计算机设计大赛-本科赛道赛题说明.pdf](../2026年广东省大学生计算机设计大赛-本科赛道赛题说明.pdf)，本项目对应如下赛题要求：

- 竞赛方向：教育方向
- 细分场景：助育（育人辅助）
- 技术路线：大模型 + RAG + 工具调用 + 智能体工作流
- 交付目标：可独立运行、具备前后端交互界面、可现场演示的智能体系统原型

## 3. 赛题要求提炼

从赛题 PDF 可提炼出 4 个必须满足的核心条件：

1. 必须选定一个真实且具体的教育场景痛点。
2. 必须实现基于领域知识的 RAG 能力，并构建专业知识库。
3. 必须实现至少一种外部工具或 API 调用，并融入智能体决策流程。
4. 必须形成完整系统，包括智能中枢、知识库、工具调用、后端服务和前端交互界面。

本项目已经围绕这 4 点完成系统设计与工程落地。

## 4. 场景定义与问题陈述

### 4.1 选题场景

WarmStudy 聚焦教育方向中的“助育”场景，即围绕学生心理健康、情绪表达、家庭沟通和早期风险识别，提供一个低门槛、可持续、可追踪的智能支持系统。

### 4.2 真实痛点

当前学校和家庭场景中普遍存在以下问题：

- 学生有情绪压力，但缺少随时可用、低压力的表达渠道
- 家长难以及时发现情绪变化，只能在冲突或异常后被动介入
- 学校心理支持资源有限，难以覆盖高频、轻量、连续性的陪伴需求
- 心理知识分散，使用者难以快速获得适合当前情境的建议

### 4.3 项目目标

本项目不是医疗诊断系统，而是一个教育与家庭场景下的心理关怀辅助智能体，目标是：

- 为学生提供安全、温和、可持续的日常情绪陪伴
- 为家长提供结构化的观察依据和沟通建议
- 为后续学校介入、家庭支持与人工辅导预留清晰接口

## 5. 核心叙事

可以用一句话概括本项目：

> 用大模型和知识库，把“学生表达不出来、家长看不出来、学校跟不上”的心理支持断层，变成一个可观察、可沟通、可干预的连续闭环。

完整叙事链如下：

1. 学生先通过日常打卡、测评和对话表达当前状态。
2. 智能体结合心理知识库与风险规则，给出陪伴式回应和知识支持。
3. 系统将结果结构化，生成趋势、建议、报告和预警。
4. 家长获得面向沟通和支持的可执行信息，而不是抽象结论。
5. 管理端可维护知识库、检索效果和系统状态，支持持续优化。

## 6. 系统方案

## 6.1 总体架构

系统由四层构成：

- 交互层：基于微信小程序技术实现的学生端 App、家长端 App，配套网页管理页面
- 服务层：`8000` 网关负责 App 接口与演示入口
- 智能体层：`5177` 服务负责 Agent、RAG、检索、对话和状态接口
- 数据与知识层：心理知识库、会话状态、报告数据、预警数据

## 6.2 架构要点

- 大模型作为智能中枢，承担理解、生成、陪伴式反馈与报告总结
- RAG 作为专业增强层，负责心理知识内容检索与回答依据补充
- 工具调用式服务整合负责把情绪识别、风险判断、报告生成、记录写入等功能纳入统一工作流
- 前后端解耦，支持比赛演示与后续部署扩展

## 6.3 与赛题要求的映射

| RAG | 已建设心理知识库接口与后台管理页面 |

| 可运行 Demo | 本地已可启动与演示，关键接口链路已打通 |

## 7. 功能设计

## 7.1 学生端

- 手机号或微信方式登录
- 心理测评与情绪打卡
- AI 对话陪伴
- 心理知识内容浏览
- 历史状态与记录查看

## 7.2 家长端

- 家长登录
- 通过二维码或绑定流程关联学生
- 查看孩子状态、打卡与心理报告
- 查看风险预警并标记处理
- 获取 AI 家庭沟通建议

## 7.3 管理与 RAG 端

- 上传和管理知识文档
- 检索测试与结果验证
- Agent 对话调试
- 系统状态与知识库状态查看

## 9. 关键技术实现

### 9.1 大模型

系统支持通过 API 接入成熟大模型，当前工程统一使用：

- 通义千问 / DashScope

这种方案符合赛题中“可使用开源模型或 API 调用模型”的要求。当前仓库已统一收敛为 Qwen / DashScope 一条模型接入路径，配置更简单，也更适合比赛现场保证稳定性。

### 9.2 RAG 知识库

系统提供心理知识检索接口和网页知识库管理能力，用于支撑：

- 心理知识问答
- 类别化知识浏览
- 领域增强回答
- 后续知识导入与精细化维护

### 9.3 工具调用与工作流

本项目的“工具调用”并不局限于单一函数，而是通过服务能力融入智能体闭环，包括：

- 心理知识检索
- 情绪状态记录
- 风险预警处理
- 家长绑定与孩子关系建立
- 心理报告生成与查询

这符合赛题要求中“外部工具或 API 调用进入推理与决策流程”的定义。

### 9.4 安全与边界

项目定位为教育支持与心理关怀辅助工具，不做医疗诊断结论。系统强调：

- 角色边界清晰
- 高风险内容需要提示人工介入
- 家长侧展示以支持建议为主，避免误判式标签化输出

## 10. 特色创新

本项目的创新点主要体现在 4 个方面：

1. 不是泛聊天机器人，而是聚焦“助育”的心理关怀智能体。
2. 不是单点问答，而是打通学生表达、家长理解、系统预警的连续闭环。
3. 不只做对话，还把知识检索、报告、预警、绑定等能力工程化整合到一套 Demo 中。
4. 具备管理后台和部署能力，更符合比赛对“完整系统原型”的要求。
