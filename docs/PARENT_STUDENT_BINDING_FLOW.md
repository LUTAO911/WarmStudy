# 家长-学生绑定流程说明

## 目标

本流程用于让家长端展示真实绑定孩子的数据，避免家长端默认展示演示假数据。

## 当前实现

1. 学生手机号登录后，后端按手机号生成固定的 9 位孩子 ID。
2. 学生端“成长中心”展示孩子 ID，并提供复制按钮。
3. 家长手机号登录后，如果尚未绑定孩子，家长首页只展示绑定引导。
4. 家长输入 9 位孩子 ID 后，后端写入双向绑定关系：
   - `parent_id -> child_id`
   - `child_id -> parent_id`
5. 绑定成功后，家长端按当前孩子 ID 拉取：
   - 综合报告
   - 心理测评报告
   - 打卡状态
   - 风险预警
   - 家长 AI 对话上下文

## 演示口径

学生完成心理测评后，系统生成风险等级、状态摘要和成长建议；家长绑定该学生后，可在家长端同步查看综合报告和测评报告。当风险项达到阈值时，系统会生成家长端预警提醒，形成测评、分析、反馈、预警的闭环。

## 后续生产化建议

- 将用户、绑定、报告、预警从内存数据迁移到数据库。
- 绑定孩子 ID 时增加学生端确认或手机号后四位校验。
- 给家长报告详情接口增加绑定权限校验。
- 为多个孩子绑定增加切换、解绑和审计记录。
