# Competition Submission Checklist

This checklist is aligned to the 2026 Guangdong University Computer Design Competition brief for the large model and agent innovation track.

## 1. Mandatory Submission Items

- [ ] project information summary form
- [ ] intelligent agent prototype project report
- [ ] demo video within 3 minutes
- [ ] defense PPT
- [ ] runnable source code
- [ ] clear `README.md` deployment document
- [ ] dependency environment configuration
- [ ] model source / API / weight acquisition note
- [ ] detailed deployment note

## 2. Recommended Repository Documents

- [x] `README.md`
- [x] `docs/DEPLOYMENT_RUNBOOK.md`
- [x] `docs/MODEL_SELECTION.md`
- [x] `docs/COMMERCIALIZATION_PLAN.md`
- [x] `docs/OPEN_SOURCE_AND_COMPONENTS.md`
- [x] `docs/MODEL_SOURCE_AND_DEPLOYMENT_NOTE.md`
- [x] `docs/PROJECT_OVERVIEW_FORM.md`
- [x] `docs/PPT_AND_DEMO_OUTLINE.md`

## 3. Content Checklist For Project Report

The project report should clearly include:

- [ ] project background
- [ ] problem and pain point definition
- [ ] selected competition direction and sub-scene
- [ ] product positioning and target users
- [ ] system architecture diagram
- [ ] core modules and technical route
- [ ] large model usage explanation
- [ ] RAG implementation explanation
- [ ] tool calling / workflow explanation
- [ ] frontend and backend integration explanation
- [ ] deployment process
- [ ] test or demo result
- [ ] innovation highlights
- [ ] limitations and future work

## 4. Technical Requirement Self-Check

- [ ] a mainstream large model is used as the intelligence core
- [ ] RAG is implemented with domain knowledge retrieval
- [ ] at least one external tool or API call is integrated into the workflow
- [ ] the system includes backend service and frontend interaction
- [ ] the agent has a clear role definition and domain boundary
- [ ] the system can present a stable interactive demo

## 5. On-Site Demo Readiness

- [ ] backend can start successfully
- [ ] frontend can connect to backend
- [ ] core demo path is rehearsed
- [ ] API keys are prepared and valid
- [ ] local `.env` is configured
- [ ] backup demo video is available
- [ ] architecture diagram and workflow diagram are ready in PPT
- [ ] one person can deploy, one person can present, one person can answer technical questions

## 6. Open Source And Compliance Check

- [ ] open source frameworks and components are listed
- [ ] license types are noted
- [ ] whether source code was modified is stated
- [ ] whether open source data / open source model is used is stated
- [ ] whether third-party commercial APIs are used is stated

## 7. Final Packaging Suggestion

Recommended final delivery structure:

```text
submission/
|- 概要表
|- 项目报告
|- 答辩PPT
|- 演示视频
|- 源代码
|- README部署说明
|- 开源代码与组件使用情况
|- 模型来源与部署说明
```
