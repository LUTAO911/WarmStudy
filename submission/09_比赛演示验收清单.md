# 比赛演示验收清单

本清单用于赛前最后一轮验收，目标是证明系统不是静态页面或普通聊天机器人，而是具备学生端、家长端、后台管理、RAG 知识库和云端部署能力的完整智能体原型。

## 1. 演示环境

- 正式接口域名：`https://wsapi.supermoxi.top`
- 对外入口：`443 -> Nginx/Caddy -> 8000`
- 内部 Agent / RAG 服务：`5177`，仅本机或内网访问
- 前端形态：基于微信小程序技术实现的 App 应用前端
- 大模型：通义千问 / DashScope
- RAG 文档：优先使用 `docs/COMPETITION_PROJECT_DESCRIPTION.md`

## 2. 真机通信验收

- [ ] 腾讯 miniapp / super app 后台已配置 request 合法域名：`https://wsapi.supermoxi.top`
- [ ] 腾讯 miniapp / super app 后台已配置 uploadFile 合法域名：`https://wsapi.supermoxi.top`
- [ ] 腾讯 miniapp / super app 后台已配置 downloadFile 合法域名：`https://wsapi.supermoxi.top`
- [ ] 如启用长连接，已配置 websocket 合法域名：`wss://wsapi.supermoxi.top`
- [ ] 域名配置后已重新打包 APK
- [ ] 真机安装包可完成登录、聊天和报告查看

## 3. 服务器接口验收

赛前至少验证以下接口：

```bash
curl -fsS https://wsapi.supermoxi.top/api/health
curl -fsS https://wsapi.supermoxi.top/api/gateway/status
curl -fsS "https://wsapi.supermoxi.top/api/agent/psychology/categories?user_type=student"
```

如使用命令行验证登录与对话，建议记录返回摘要或截图：


## 4. RAG 验收

- [ ] 后台可查看 RAG 服务状态
- [ ] 可上传或同步测试文档
- [ ] `/api/gateway/rag/index/info` 返回文档或片段数量
- [ ] `/api/gateway/rag/search` 能返回相关知识片段
- [ ] `/api/gateway/rag/chat` 能基于知识片段生成回答
- [ ] 演示时能说明“检索结果来自知识库，而不是纯模型自由发挥”

建议演示问题：

```text
暖学帮如何解决学生表达困难、家长理解不足和学校支持有限的问题？
```

## 5. 3 分钟演示路线

1. 20 秒：说明场景痛点，强调青少年心理关怀与家校协同。
2. 45 秒：学生端输入压力与睡眠困扰，展示 AI 安抚和行动建议。
3. 35 秒：学生完成测评，展示状态摘要、风险提示和成长建议。
4. 40 秒：家长端查看孩子状态和报告，并向 AI 咨询沟通方式。
5. 35 秒：后台展示 RAG 检索、问答、知识库状态和系统健康状态。
6. 25 秒：总结技术路线，大模型 + RAG + 工具调用 + 前后端闭环。

## 7. 答辩口径

- 本项目不是医疗诊断系统，而是教育场景下的心理关怀与家校协同辅助系统。
- 学生端强调情绪接住、低压力表达和适龄建议。
- 家长端强调观察线索、低冲突沟通和可执行行动。
- 后台强调知识库维护、RAG 检索验证、系统状态和部署可维护性。
- RAG 的价值在于把领域知识引入回答，降低泛聊天和凭空生成的风险。
