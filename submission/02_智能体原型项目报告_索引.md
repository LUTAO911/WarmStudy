# 智能体原型项目报告

## 1. 项目名称

WarmStudy（暖学帮）：面向青少年心理关怀的教育智能体原型系统

## 2. 赛题对应

本项目对应 2026 年广东省大学生计算机设计大赛教育方向“助育”场景，围绕学生心理健康、家庭沟通与数字化支持展开。

## 3. 项目背景

青少年在学习、家庭和社交环境中容易积累情绪压力，但很多学生缺少低门槛表达渠道，家长又难以及时发现长期变化趋势。学校心理支持资源有限，难以覆盖高频、轻量、持续性的陪伴需求。

因此，本项目希望构建一个面向教育场景的心理关怀智能体，用大模型、RAG 和工具调用能力，把“表达、观察、支持、预警”连成闭环。

## 4. 核心问题

项目主要解决以下问题：

1. 学生没有便捷、连续、低压力的情绪表达入口
2. 家长难以理解长期情绪变化和沟通重点
3. 学校缺少轻量化数字工具支撑心理关怀流程

## 5. 项目定位

WarmStudy 定位为教育与家庭场景下的心理关怀辅助智能体，不是医疗诊断系统。它面向“助育”而不是“治疗”，强调日常支持、结构化观察和风险提醒。

## 6. 系统方案

系统分为四层：

1. 交互层：基于微信小程序技术实现的学生端 App、家长端 App、统一管理员后台
2. 网关层：`8000` 负责唯一网页入口和前端 API 聚合
3. 智能体层：`5177` 提供 Agent、RAG、检索和对话 API
4. 数据层：知识库、打卡、报告、预警、活动日志等数据

## 7. 核心技术

### 7.1 大模型

当前项目统一使用：

- 通义千问 / DashScope

### 7.2 RAG

项目构建了心理知识库，支持：

- 知识检索
- RAG 问答
- 文档上传入库
- 文档更新、删除和重置

### 7.3 工具调用与工作流

项目通过服务式工具整合实现：

- 学生情绪打卡
- 心理测评
- 报告生成
- 9 位孩子 ID 生成与家长绑定
- 风险预警
- 管理员后台检索和知识库管理

## 8. 功能实现

### 8.1 学生端

- 登录
- 心理测评
- 情绪打卡
- AI 对话
- 心理知识浏览

### 8.2 家长端

- 登录
- 首次登录输入 9 位孩子 ID 绑定孩子
- 查看综合报告与测评报告
- 查看预警
- AI 沟通建议

### 8.3 管理员后台

- App 后台总览
- 用户信息与登录记录
- 模型使用统计
- RAG 知识库管理
- RAG 检索与问答

## 9. 部署方式

当前系统采用双服务、单网页入口方案：

- `8000`：唯一网页后台与 API Gateway
- `5177`：内部 Agent / RAG API

Docker 部署时只暴露 `8000`。`5177` 仅作为内部服务，不再单独暴露网页界面。

## 10. 测试与验证

当前版本已完成：

- 核心接口链路修复
- 登录、9 位孩子 ID 绑定、综合报告、测评报告、预警、知识库接口联调
- 管理员后台单入口收敛
- 全量测试通过：`309 passed, 16 skipped`

## 11. 创新点

1. 聚焦教育方向“助育”，而不是泛聊天机器人
2. 打通学生表达、家长观察、系统预警的连续闭环
3. 将大模型、RAG 和业务工具能力整合为可演示系统
4. 采用统一管理员后台，增强工程完整性和运维可视化

## 12. 当前局限

当前版本适合比赛演示和单机部署，但正式长期运行仍需补齐：

1. 将内存态业务数据迁移到数据库
2. 增加正式登录鉴权和管理员权限体系
3. 完善日志持久化和审计机制

## 13. 后续计划

1. 引入 PostgreSQL / MySQL 存业务数据
2. 引入 Redis 存缓存和会话
3. 保留 Chroma 作为向量库
4. 完善管理员后台的数据持久化与检索能力

## 14. 结论

WarmStudy 已经具备清晰场景、完整架构、可运行代码、统一管理员后台和比赛所需说明文档，能够作为教育方向“助育”场景下的智能体原型系统参加展示和答辩。
