# WarmStudy（暖学帮）

WarmStudy（暖学帮）是一套面向青少年心理关怀与家校协同支持场景设计的教育智能体原型系统。项目围绕“学生表达—家长支持—心理知识—风险提醒—后台管理”的完整链路展开，强调在真实教育场景中提供可理解、可配置、可落地的智能支持，而不是停留在泛聊天或静态问答层面。

## 项目概述

在学校与家庭共同参与的心理关怀场景中，真正困难的往往不是“有没有模型”，而是：

- 学生是否拥有低门槛、持续性的表达入口；
- 家长是否能获得温和、可执行的沟通建议；
- 管理者是否可以统一查看模型、知识库与运行状态；
- 系统是否能根据用户年龄、阶段与近期状态进行个性化响应。

WarmStudy 正是围绕以上问题构建。它不是简单把大模型接到前端，而是通过角色分层、状态联动、RAG 检索增强和统一后台控制台，将学生端、家长端与管理员端组织成一套闭环系统。

## 设计目标

WarmStudy 的核心目标可以概括为四点：

1. 场景真实：聚焦青少年心理关怀、家庭沟通与日常支持，而非泛化对话。
2. 角色清晰：区分学生端、家长端与管理员后台的职责与交互方式。
3. 个性化支持：结合年龄、学段、打卡、测评与风险变化动态调整响应策略。
4. 可观测可管理：统一查看模型配置、知识库、活动流与系统运行状态，便于演示、部署与维护。

## 核心价值

### 1. 面向真实教育心理场景

WarmStudy 将智能体应用到具体而高频的教育支持场景中，重点解决：

- 学生情绪压力难以及时表达；
- 家长缺少温和、专业、非指责式的沟通建议；
- 学校心理支持资源有限，难以提供高频、轻量、持续性的陪伴；
- 管理者缺少统一入口来配置模型、维护知识库和观测系统。

### 2. 多角色协同，而非单点工具

系统面向三类核心角色构建协同能力：

- 学生端：提供情绪陪伴、心理测评、每日打卡、心理知识浏览与 AI 对话；
- 家长端：提供孩子状态查看、报告解读、预警提醒与 AI 沟通建议；
- 管理员后台：提供账号、模型、RAG 知识库、活动流与系统概览等统一管理能力。

### 3. 状态驱动的智能体策略

WarmStudy 不采用“所有人同一套固定 Prompt”的简单方案，而是结合：

- 年龄 / 年级 / 学段；
- 近期打卡结果；
- 测评状态；
- 风险变化趋势；
- 历史交互上下文；

动态刷新智能体策略，使回答更贴近真实用户状态与当下需求。

### 4. 后台统一、部署清晰

当前版本将管理能力收敛到统一网页后台，减少演示入口分散和部署复杂度，更适合比赛展示、答辩讲解与后续维护，也有助于提升系统的一致性与稳定性。

## 智能体个性化亮点

### 学生端智能体

学生端强调“先接住情绪，再给出适龄支持”，在表达风格与建议粒度上做了针对性区分：

- 小学阶段：短句、具体、降低抽象表达；
- 初中阶段：语气更温和，弱化说教感；
- 高中阶段：强调尊重、自主性与节奏感。

系统会结合当前心理状态动态切换支持重点，例如：

- 更偏安抚与陪伴；
- 更偏一步式建议；
- 更偏鼓励表达；
- 更偏缓解学习压力刺激。

### 家长端智能体

家长端采用独立人格与策略，不复用学生端的对话逻辑，强调：

- 结构化表达；
- 可观察的状态线索；
- 可执行的下一步沟通动作；
- 避免责备式、贴标签式、制造焦虑式建议。

### 数据联动式策略刷新

学生提交打卡、测评或出现状态变化后，系统能够刷新策略状态，使智能体输出尽量贴合当前情境，而非停留在静态设定层面。

## 当前版本核心能力

### 学生端

- 登录
- 心理测评
- 情绪打卡
- AI 对话
- 心理知识浏览

### 家长端

- 登录
- 绑定孩子
- 查看状态与报告
- 查看预警
- 获取 AI 沟通建议

### 管理员后台

- App 后台总览
- 用户与登录记录
- 模型使用统计
- 模型配置管理
- RAG 知识库上传、更新、删除、重置
- RAG 检索与问答
- 近期活动流与系统概览

## 系统架构

### 对外统一入口：`8000`

- 统一网页入口；
- API Gateway；
- 管理员后台控制台；
- 聚合 App 后台、RAG、模型使用与账号数据。

### 内部服务：`5177`

- Agent / RAG API 服务；
- 负责检索、问答、知识库与模型配置等能力；
- 当前版本不再独立对外提供单独 Web UI。

## 技术路线

- 大模型：Qwen / DashScope
- 检索增强：RAG
- 向量数据库：ChromaDB
- 后端框架：Python / Flask
- 前端形态：基于微信小程序技术实现的 App 前端工程
- 管理后台：统一网页控制台

## 快速启动

### 本地启动

```powershell
cd agent
.\start_all.ps1
```

启动后访问：

- 管理员后台：`http://localhost:8000/`

### Docker 启动

```powershell
docker compose up --build
```

默认对外暴露端口：

- `8000:8000`

## 关键环境变量

```env
CHAT_MODEL=qwen

RAG_AGENT_URL=http://localhost:5177

LOG_LEVEL=INFO
```
