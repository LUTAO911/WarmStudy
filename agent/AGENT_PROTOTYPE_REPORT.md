## 摘要

本项目针对K12教育场景中教师教学负担重、学生个性化辅导难以实现等痛点问题，设计并实现了一个基于大语言模型的智能教育助手系统——"暖学帮"。系统集成了检索增强生成（RAG）技术、工具调用框架和工作流管理机制，提供助教、助学、助评、助管、助育五大核心功能模块，实现了对教学全流程的智能化辅助。实验表明，该系统能够有效提升教学效率，为学生提供个性化的学习支持。

**关键词**: 智能教育助手；检索增强生成；大语言模型；工具调用；教育智能化

## 一、背景与问题定义

### 1.1 项目背景

随着人工智能技术的快速发展，教育领域正在经历深刻的变革。传统的教学模式面临以下挑战：

1. **教学资源不均衡**: 优质教育资源集中在少数地区和学校
2. **个性化辅导难实现**: 教师难以针对每个学生的特点进行针对性辅导
3. **作业批改负担重**: 教师花费大量时间在重复性的批改工作上
4. **学习效果评估难**: 缺乏科学的、多维度的学习评价体系

### 1.2 核心问题

本项目聚焦以下核心问题：

| 问题 | 现状 | 痛点程度 |
|------|------|----------|
| 作业批改耗时 | 教师手工批改，效率低 | ⭐⭐⭐⭐⭐ |
| 答疑资源有限 | 一对一辅导成本高 | ⭐⭐⭐⭐ |
| 学习路径单一 | 统一教学难以因材施教 | ⭐⭐⭐⭐ |
| 评价体系单一 | 缺乏过程性评价 | ⭐⭐⭐ |

### 1.3 解决方案概述

我们提出了"暖学帮"智能教育助手系统，通过AI技术辅助教学各个环节：

### 2.2 核心模块设计

#### 2.2.1 智能体核心 (Agent Core)

```python
class Agent:
    def chat(self, message, session_id, use_rag=True, use_tools=True):
        # 1. 意图识别
        intent = self.classify_intent(message)

        # 2. 知识检索 (RAG)
        if use_rag:
            context = self.retrieve_context(message)

        # 3. 工具调用
        if use_tools:
            tools = self.select_tools(intent, message)
            results = self.execute_tools(tools)

        # 4. 响应生成
        response = self.generate_response(intent, context, results)
        return response
```

#### 2.2.2 五大功能模块

| 模块 | 核心功能 | 技术实现 |
|------|---------|---------|
| **助教** | 作业生成、批改、答疑、课件生成 | LLM+RAG |
| **助学** | 学习路径规划、知识点讲解、练习生成 | LLM+知识库 |
| **助评** | 答案评价、反馈生成、进度追踪 | LLM+评估算法 |
| **助管** | 课程管理、资料管理、学生管理 | 数据库+LLM |
| **助育** | 心理健康评估、素质培养建议 | LLM+规则引擎 |

class QueryExpander:
    def expand(self, query):

        synonyms = {"学习": ["学", "掌握"], "问题": ["疑问", "困惑"]}
