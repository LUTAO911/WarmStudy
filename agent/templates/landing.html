System Design

系统方案不是堆料，而是按场景拆层

学生和家长从前端进入系统，统一网关聚合服务入口，Agent 策略层区分角色与状态，RAG 层提供知识增强，后台负责模型、知识库与系统观察。

核心架构

Agent + 状态 + RAG 共同工作

从意图路由、状态驱动策略、ReAct 机制，到混合搜索、线程安全记忆与工作流引擎，WarmStudy 更接近“场景智能体系统”而不是单点大模型应用。

  • 学生端优先陪伴与低压表达
  • 家长端优先结构化建议与沟通支持
  • 后台帮助评委直接看懂模型与业务如何串联
模型链路设计 通过 LangChain 协调大模型与 Embedding 模型，把自然语言理解和知识表示两条路线都讲清楚。
双端协同设计 不是学生和家长各有一个 AI 就结束，而是通过绑定关系把表达、理解、观察和提醒连成真实闭环。
Key Flows

把技术流程和业务流程都讲清楚

这部分不是纯技术说明，而是答辩时最容易帮助评委快速理解“系统为什么成立”的关键证据。

RAG · 提问前

先把知识整理好，回答才有依据

心理研究报告被切分成片段，再经由 text-embedding-v3 转成向量写入数据库，这一步决定了系统是否真正拥有可检索、可调用的知识底座。

RAG · 提问后

不是直接生成，而是先召回、再重排、再回答

学生问题先向量化，系统召回多个相关片段，经由 Cross-Encoder 重排，再把最相关内容送给大模型，提升回答可靠性与贴合度。

Business Flow

评测不是结束，而是支持链路的开始

学生完成评测后，系统生成专业性心理报告，并同步给家长端查看与接收建议，把“测试结果”变成“可以被理解和行动的支持”。

State Awareness

理解状态，而不只是理解句子

通过情绪识别、打卡结果和历史上下文，WarmStudy 让智能体回答更贴近学生此刻的心理状态，而不是输出一套固定模板。
